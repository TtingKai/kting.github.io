Step1 - Reconnaisance



會發現有一個wp-admin的網頁


當我們使用靶機IP去搜尋時，會跑出Server Not Found



Step2 - Information Gathering

這時我們想起靶機的Technical Information敘述說，我們需要到/etc/hosts修改此設定檔，需要把靶機IP對應到一個domain name

修改完後，重啟apache2


再次使用靶機IP連線時，就會連線成功


WPScan

是針對於基於WordPress架設的網站進行弱點掃描的工具
除了弱點掃描其實也是可以進行滲透測試，簡單來說，是一個針對於 Wordpress 還滿周到的安全測試工具

-u [target]；--url [target] - 指令目標 url 路徑(網址 or IP)
-e [option(s)]；--enumerate [option(s)] - 開啟枚舉功能

enumerate Options:

u - 枚舉帳號名，默認從 1-10



開一個txt檔案儲存所有用戶名稱


在靶機CLUE的敘述中，提示我們可以透過rockyou.txt檔案抓取出字串有k01的密碼，並儲存到passwords.txt檔案中


-U [Username/file]；-usernames [file] - 指定密碼或爆破用的密碼字典
-P [password/file]；-passwords [file] - 指定密碼或爆破用的密碼字典

爆破結果，我們會看到一個mark的username和helpdesk01的password


我們回到帳密登入網頁，用此帳號密碼登入


上網搜尋會發現wordpress正常的儀表板資訊如下:

比對一下，靶機的 Dashboard 會發現多出有一個Activity monitor，上網搜尋一下這是做甚麼的，會發現它是一個一個完整、易於使用且獲得良好支援的 WordPress 活動日誌外掛程式。



Step3 - Exploitation

searchsploit

是命令列(command line)形式的Exploit-DB，它能夠離線時使用。

Exploit-DB

列出了很多已知的漏洞，加速滲透測試工程師尋找漏洞時不需要自己編寫程式碼，只需要修改部分的參數，抑或是按照步驟一步一步就能夠進行攻擊。

搜尋完，會發現到有一個RCE的腳本可以使用


看一下此腳本如何使用，可以看到Usage的地方有教我們如何使用

Step4 - Privilege Escalation

方法一


方法二


透過下面指令，將指令環境變成 bash 環境


查看graham、jens、mark、sarah這四位使用者資料夾底下有哪些資料

graham

jens
會發現到一個backups.sh的檔案，使用cat指令進一步查看檔案在做什麼

mark
會發現到有一個stuff的資料夾，會發現此資料夾裡面有一個things-to-do.txt檔案，裡面是帳號和密碼

sarah


使用things-to-do.txt檔案找到的帳號和密碼，切換使用者


查看sudo可以執行的命令，會發現變成jens使用者，她有權限執行backups.sh檔案


先到jens資料夾中，使用echo指令把/bin/bash寫入到backups.sh檔案中，讓等等執行backups.sh檔案時可以為bash環境



查看jens管理者可以使用哪些權限，發現改為root可以使用nmap提權


'os.execute("/bin/bash")' -透過 Lua 的系統呼叫功能啟動一個 Bash Shell 命令列的動作

Nmap Scripting Engine (NSE)

是 Nmap 的一個強大功能，它允許任何人編寫並分享各種不同類別的腳本，在最基本的層次上，一個 NSE 腳本可以執行網路探索 (Discovery) 查詢，獲取額外的 whois 資訊、SNMP 查詢細節、以及 NFS/SMB 共享資訊，這些探索腳本可以產生「潛在問題裝置」清單，進一步進行版本檢測等調查


提權成功!!
